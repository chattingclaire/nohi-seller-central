# Nohi Seller Central

> [English Version](README.md)

> AI 驱动的卖家平台，集品牌管理、智能商品目录、渠道管控与数据分析于一体。

基于 **Next.js 16**、**React 19**、**Tailwind CSS 4** 和 **Radix UI** 构建。

---

## 品牌画像 — AI 辅助品牌配置

品牌画像模块是整个品牌配置的核心枢纽。不同于传统的多页表单，它采用了 **AI 对话 + 表单向导** 的交互模式 — 商家在左侧上传品牌资料，Nohi 自动解析并填充右侧的结构化表单。

### 核心功能

#### 1. 新手引导卡片
商家首次进入页面时，会看到一张手写风格的欢迎卡片，简要说明工具的使用方式：
- 在下方输入框中粘贴或上传品牌资料
- Nohi 会自动阅读并填充所有表单步骤
- 也可以跳过，直接手动编辑表单

![初始状态](docs/brand-context/01-initial-onboarding.png)

#### 2. 文件上传 — 粘贴、附件或拖拽
Claude 风格的组合输入框支持多种文件添加方式：
- **粘贴**：直接 `Ctrl/Cmd+V` 将文件粘贴到输入区
- **附件**：点击回形针按钮打开文件选择器
- **支持格式**：PDF、PPT、DOC、图片（PNG/JPG/SVG）、CSV、XLSX、TXT

发送前，文件会以可移除的标签形式显示在输入区内。

![文件标签](docs/brand-context/02-file-chips-pending.png)

#### 3. AI 一键填充 — 同时填写所有步骤
文件发送后，Nohi 会解析内容并 **同时填充全部 5 个步骤**：

| 步骤 | 填充内容 |
|------|---------|
| 基本信息 | 核心品类、客单价区间、购买类型、人群/场景/意图标签 |
| 品牌故事 | 150 词品牌叙事 + 创始人寄语 |
| 视觉风格 | 风格标签（极简、北欧风等） |
| 防护规则 | 排除人群、禁用场景、屏蔽关键词 |
| 履约信息 | 处理时效、准时率、退货指标 |

> **原则：尽量保留商家原始表述。** Nohi 在填充表单字段时，会优先使用商家自己的语言和表述方式 — 品牌故事、创始人寄语、标语等内容会尽可能原样提取，而不是由 AI 重写或改写。确保品牌声音的真实性和一致性。

AI 的回复会给出详细的填充报告：
- **已填充部分**：附带直达链接，方便逐步审核
- **未完成部分**：提示需要补充哪些资料

![自动填充](docs/brand-context/03-auto-filled-response.png)

#### 4. 智能对话逻辑

对话系统能根据不同场景做出智能响应：

| 用户操作 | Nohi 的响应 | 表单影响 |
|---------|------------|---------|
| **上传文件（PDF/PPT/图片等）** | 解析全部文件，展示已填充和缺失内容的详细报告，建议补充上传 | 一次性自动填充全部 5 个步骤 |
| **纯文本（尚未上传文件）** | "上传品牌文件，Nohi 会帮你自动填充表单。也可以直接编辑右侧表单。" | 不改动任何字段 |
| **纯文本（已上传过文件）** | "可以用 @ 引用已上传的文件，或直接在右侧编辑对应字段。" | 不改动任何字段 |
| **在输入框中输入 `@`** | 弹出文件选择下拉菜单，点击文件名即可插入 `@文件名` 引用 | 不改动任何字段 |

**场景 A：纯文字消息（未上传文件）** — Nohi 引导用户上传文件，不触碰任何表单字段：

![纯文字回复](docs/brand-context/08-text-only-no-files.png)

**场景 B：@ 引用文件** — 上传文件后，输入 `@` 即弹出已上传文件列表，点击插入引用：

![@ 文件引用](docs/brand-context/09-at-mention-file-picker.png)

#### 5. 可折叠文件列表
上传完成后，文件以紧凑的折叠头部显示：
- 默认单行展示 `"📎 3 个文件 >"` — 占用最小纵向空间
- 点击展开查看每个文件的名称、类型和大小
- 切换步骤时文件列表保持不变

#### 6. 分步向导（右侧面板）
每个步骤保留了原始的表单设计：
- **TagInput 标签输入**组件：支持标签云、自定义标签和"生成更多"浮层
- **按钮选择器**：用于品类、客单价区间、购买类型的选择
- **文本域**：带字数统计，用于品牌故事和创始人寄语
- **指标输入**：用于履约 SLA 数值
- **下拉选择**：用于退款时效
- 左右面板独立滚动 — 右侧表单与左侧对话互不影响

#### 7. 中英双语支持
所有文案 — 引导卡片、占位文字、AI 回复、表单标签 — 均通过 `useLanguage()` Hook 支持中英文切换，语言会根据用户设置自动检测。
